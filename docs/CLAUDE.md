# Documentation Agent Rules

```text
[TBG | Docs Rules | scope: docs]
```

## Purpose

Docs are operational inputs. They define what future agents and humans will do.

## Required for serious handoff docs

Include:

- context banner
- repo and branch
- sprint or PR
- scope and forbidden scope
- changed files or target files
- validation commands
- gaps and risks
- next command

## Writing style

Keep docs direct, actionable, and bounded. Prefer contracts, checklists, and exact paths over vague principles.

## Do not

- Claim runtime proof from documentation alone.
- Leave ambiguous branch or PR identity.
- Hide skipped checks.
