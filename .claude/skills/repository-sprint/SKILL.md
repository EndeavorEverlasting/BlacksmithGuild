# Skill: Repository Sprint

## Trigger

Repository intake, sprint selection, interrupted work recovery, Git/worktree/PR lifecycle, or harness evolution.

## Capability dependencies

- `repository-evidence`
- `proof-and-checkpointing`

## Inputs

- repository path or remote;
- user mission and safety constraints;
- branch/PR or plan when known.

## Procedure

1. Run compact preflight.
2. Read `AGENTS.md`, `CODEBASE_MAP.md`, and the routing manifest.
3. Harvest current product, harness, validation, runtime, and unresolved evidence.
4. Rank bounded candidates using floor-before-furniture and shared-contract-first rules.
5. State lane, scope, forbidden scope, dependencies, artifacts, validators, and proof ceiling.
6. Execute the first safe sprint.
7. Validate, commit, push, and open/update the PR when available.
8. Generate a sprint capsule for any continuation.

## Outputs

- evidence-backed sprint decision;
- useful tracked changes or exact blocker;
- validation record;
- commit/PR state;
- machine-readable handoff.

## Forbidden scope

- destructive Git cleanup of unknown work;
- game/save mutation without runtime-proof routing;
- unbounded repository census documents;
- proof claims above observed evidence.
